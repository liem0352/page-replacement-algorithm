# 页面置换算法实验实训

本项目实现了两种经典的页面置换算法：
1. FIFO（先进先出）算法
2. LRU（最近最久未使用）算法

## 项目结构
- `main.c` - 主程序文件，包含用户交互界面
- `fifo.c` - FIFO算法实现
- `lru.c` - LRU算法实现
- `page_replacement.h` - 头文件，包含数据结构和函数声明
- `page_replacement.exe` - 编译后的可执行文件

## 算法说明

### FIFO（先进先出）算法
FIFO算法是一种简单的页面置换算法，它总是淘汰最先进入内存的页面，即选择在内存中驻留时间最久的页面予以淘汰。

### LRU（最近最久未使用）算法
LRU算法基于局部性原理，淘汰最近最久未被访问的页面。该算法为每个页面设置一个访问时间戳，每次访问页面时更新时间戳，置换时选择时间戳最小的页面。

## 编译方法
```bash
gcc *.c -o page_replacement.exe
```

## 运行方法
双击运行 `page_replacement.exe` 或在命令行中执行：
```bash
.\page_replacement.exe
```

## 使用说明
1. 输入页面引用串的长度
2. 输入页面引用串（用空格分隔）
3. 输入内存帧数
4. 选择要演示的算法：
   - 1: FIFO算法
   - 2: LRU算法
   - 3: 比较两种算法
   - 0: 退出程序

## 示例
页面引用串：1 2 3 4 2 1 5
内存帧数：3

程序会显示每一步的页面访问过程，并统计缺页次数和缺页率。

## 实验结果分析
通过对比两种算法的缺页次数和缺页率，可以分析不同页面置换算法的性能差异。
一般来说，LRU算法的性能优于FIFO算法，因为LRU考虑了页面的访问历史。
